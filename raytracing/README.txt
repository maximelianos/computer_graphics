Машграф, задание на трассировку лучей. 6 семестр, кафедра ИИТ.

В основном задание делалось по гайдам из интернета, только под конец выполнения
я понял, что на лекциях большая часть необходимой теории разбиралась. Первый гайд
который я нашел в гугле - ray tracing in one weekend (the next week, the rest of
your life). Это подробный гайд с кодом который можно копировать. Способ
построения картинки в нем абсолютно не совпадает с первой частью задания, где
требуется гораздо более быстрый алгоритм. Возможно, этот способ является
трассировкой путей.

Содержание папок

all - библиотека чтения/записи bmp (arash_partow, с гитхаба), скрипты на питоне
по генерации сетки для изображений букв

final - код всех сцен, слитый в один файл с помощью namespace, очень большой
и непонятный. Инструкция по запуску - final/README.txt

final_src - читаемый код разных сцен перед сдачей.
2_ready2 - 1 (с буквами) сцена, 2 сцена (с шариками и беск. плоскостью)
hard_ready1 - только 1 сцена
base_ready2 - только 2 сцена
ready3 - только 3 сцена (медленный рандомный алгоритм)
letters_more - сцена с надписью "она снится мне по ночам", исходный файл
скопирован с 2_ready2. Добавленные буквы находятся в all/letters_more

timeline - стадии написания кода.
base1 - быстрый алгоритм, без треугольных мешей
base2 - добавление простых мешей, я еще тогда не строил сетку
my1 - медленный алгоритм из ray tracing in one weekend
my2 - изменил сцену, попробовал кубик
original_whitted - код со scratchpixel, на котором я понял как устроен быстрый
алгоритм


***
Для запуска:
Скопировать в одну папку all/letters, all/bitmap_image.hpp, all/makefile,
all/panorama*.bmp, all/portrait.bmp, final_src/2_ready2.cpp (или другой файл
из этой папки)
Подправить makefile, файл с кодом
make && ./exe
